# 第一部分：Rust 异步编程指南

本书的这一部分是关于异步 Rust 的教程风格指南。它的目标读者是 Rust 异步编程的新手。无论你是否在其他语言中做过异步编程，它都应该很有用。如果你有经验，你可以跳过第一节，或者浏览一下作为复习。你可能还想尽早阅读这篇[与其他语言异步的比较]()。

## 核心概念

我们将从讨论不同的[并发编程](concurrency.md)模型开始，使用进程、线程或异步任务。第一章将涵盖 Rust 异步模型的基本部分，然后在[第二章](async-await.md)中深入探讨异步编程的细节，我们将介绍 async 和 await 编程范式。我们在[下一章](more-async-await.md)中涵盖了更多异步编程概念。

异步编程的主要动机之一是更高效的 IO，我们在[下一章](io.md)中对此进行了介绍。我们在同一章中也详细介绍了*阻塞*。阻塞是异步编程中的一个主要危险，线程会被一个同步等待的操作（通常是 IO）阻塞，无法继续执行。

异步编程的另一个动机是它促进了[并发代码的抽象和组合](concurrency-primitives.md)的新模型。涵盖这些之后，我们将继续讨论并发任务之间的[同步](sync.md)。

有一章是关于[异步编程工具](tools.md)的。

最后几章涵盖了一些更专业的主题，从[异步析构和清理](dtors.md)开始（这是一个常见的需求，但由于目前没有很好的内置解决方案，所以是一个比较专业的主题）。

指南的接下来的两章详细介绍了[futures](futures.md)和[runtimes](runtimes.md)，这是异步编程的两个基本构建块。

最后，我们涵盖了[定时器和信号处理](timers-signals.md)以及[异步迭代器](streams.md)（又名 streams）。后者是我们如何对异步事件序列进行编程（参考：使用 futures 或异步函数表示的单个异步事件）。这是一个语言正在积极开发的领域，可能还有些不完善。
